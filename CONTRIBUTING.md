# Contributing to This Repository

Thank you for your interest in contributing! This repository serves as a professional portfolio showcasing DevOps best practices and examples.

## How to Contribute

### Reporting Issues

If you find any issues with the examples or documentation:

1. Check if the issue already exists
2. Create a new issue with a clear description
3. Include steps to reproduce if applicable
4. Add relevant labels

### Suggesting Enhancements

Have ideas for new examples or improvements?

1. Open an issue describing your suggestion
2. Explain the use case and benefits
3. Be open to discussion and feedback

### Pull Requests

When submitting pull requests:

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/amazing-example`)
3. Make your changes following the style guide
4. Test your changes thoroughly
5. Commit with clear, descriptive messages
6. Push to your fork
7. Open a pull request with a clear description

## Code Standards

### Terraform

- Use consistent formatting: `terraform fmt`
- Validate configurations: `terraform validate`
- Include meaningful variable descriptions
- Use variables for customizable values
- Document modules with README files

### Kubernetes

- Follow YAML best practices
- Include resource limits and requests
- Use health checks (liveness/readiness probes)
- Implement security contexts
- Add meaningful labels and annotations

### Docker

- Use multi-stage builds when appropriate
- Run containers as non-root user
- Minimize image layers
- Include health checks
- Use specific base image versions

### CI/CD Pipelines

- Use meaningful stage and job names
- Include comments for complex logic
- Implement proper error handling
- Use variables and parameters
- Follow DRY principles

## Documentation

- Update README files when adding examples
- Include prerequisites and usage instructions
- Provide clear explanations of concepts
- Add comments for complex configurations
- Keep documentation up-to-date
